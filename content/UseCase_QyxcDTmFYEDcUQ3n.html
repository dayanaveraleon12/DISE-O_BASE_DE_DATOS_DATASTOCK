Use Case Note
■ Workflow
    • // Write down briefly how user perform the work
    • El Empresario Admin de Inventario o empleado ingresa a "Consultar Listas".
    • El sistema muestra las listas gestionadas de los últimos 5 meses o las listas pendientes por gestionar.
    • El Admin de Inventario va a seleccionar en la lista pendiente por gestionar los artículos que están disponibles o tienen alguna novedad.
    • El Admin. de Inventario puede rechazar un producto y dejar una descripción del por qué no lo pudo entregar en la lista.
    • Cuando el Admin. de Inventario o Empleado gestiona las listas, el sistema cambia el estado de las listas a "Gestionado".
    • Una vez gestionada la lista, el sistema descuenta de forma automática la cantidad de productos entregados en la lista.
    • Los productos que fueron rechazados en la lista, no serán descontados.
    • El sistema guarda las listas gestionadas y las clasifica de acuerdo a su fecha y hora.
■ Business Logic
    • // Write down what user expect the system to react upon certain condition (e.g. low inventory alert level)
    • Se espera que se puedan visualizar las listas pendientes y gestionadas.
    • Se espera que las listas se puedan clasificar por fecha y hora.
    • Se espera que el sistema no borre las listas a no ser que se cumplan 5 meses.
    • Se espera ver los productos gestionados y los productos rechazados.
    • Se espera que los productos rechazados permitan agregar una descripción del por qué fueron rechazados.
■ Decisions
    • // Write down the decisions made during the meeting (e.g. Must allow accessing from mobile devices)
    • Se decidió que debe existir un estado de lista para que los empleados sepan si el Admin. de Inventario a gestionado las listas o no.
    • Se decidió que el sistema va a guardar las listas de hasta al menos 5 meses en caso de necesitar algún tipo de seguimiento.
    • Se decidió que las listas deben tener y clasificarse por: fecha, hora y empleado que solicitó la lista.
■ Follow-up
    • // Write down the items that should follow-up in the coming meeting
    • Se discutirá si el empleado podría modificar o eliminar la lista una vez enviada al Admin de Inventario.

    2.1. Fecha

4. SYSTEM Muestra las listas que cumplen con los criterios.
5. EMPRESARIOADMIN INVENTARIOEMPLEADO Selecciona una lista específica.
6. SYSTEM muestra el detalle:

    6.2. Cantidades
    6.3. Fecha de solicitud
    6.4. Fecha de gestión
    6.5. Usuario que gestionó
    6.6. Estado de la lista
    6.7. Número de lista
    6.8. Origen de la solicitud (Bodega o Número de Tienda)
