Use Case Note
■ Workflow
    • // Write down briefly how user perform the work
    • El Empresario crea los usuarios de los demás actores con datos personales como: nombres, apellidos, celular, contacto de emergencia, EPS y correo.
    • Tiene la opción de asignar rol al usuario a crear.
    • El Empresario crea una contraseña provisional al usuario en creación
    • Le da clic a crear usuario.
    • El sistema valida los datos ingresados y muestra un mensaje de registro exitoso con sus credenciales asignadas ó en caso contrario cual dato es incorrecto.
■ Business Logic
    • // Write down what user expect the system to react upon certain condition (e.g. low inventory alert level)
    • El sistema debe verificar que el correo electrónico no esté ya registrado
    • El sistema valida que la contraseña provisional sea alfanumerica con un caracter especial con un total de 8 caracteres oómás
    • Sí todo es correcto se crea el nuevo usuario en la base de datos y se envía correo de confirmación de registro exitoso con las credenciales al destinatario
■ Decisions
    • // Write down the decisions made during the meeting (e.g. Must allow accessing from mobile devices)
    • Se decidió que solo el actor empresario crea los usuarios de todos.
    • Al correo del destinatario creado también llega adicional sus credenciales de acceso.
■ Follow-up
    • // Write down the items that should follow-up in the coming meeting
    • Envió de notificaciones de reigistro exitoso por vía mensaje de texto o WhatsApp
