Use Case Note
■ Workflow
    • // Write down briefly how user perform the work
    • El administrador registra un ingreso de inventario
    • Añade una descripción detallada del lote / producto ingresado (proveedor, estado, observaciones)
    • Guarda la información junto con el ingreso
■ Business Logic
    • // Write down what user expect the system to react upon certain condition (e.g. low inventory alert level)
    • Asocia la descripción al movimiento de entrada/ingreso de productos
    • Permite consultar la descripción en el historial
    • Valida que la descripción no esté vacía en ingresos críticos como en "Estado" haber seleccionado "Dañado" ó "No llego por pedido" ó "Otro"
■ Decisions
    • // Write down the decisions made during the meeting (e.g. Must allow accessing from mobile devices)
    • La descripción será obligatoria en ingresos criticos
    • Debe permitir texto libre
    • Se debe poder consultar posteriormente
■ Follow-up
    • // Write down the items that should follow-up in the coming meeting
    • Definir longitud máxima de descripción
    • Evaluar uso de plantillas de descripción
    • Considerar etiquetas (ej: urgente, revisión)

Scenario
Steps
1. ADMIN INVENTARIO EMPRESARIO Accede a INGRESOS DE PRODUCTOS
2. EMPRESARIOADMIN INVENTARIO Ingresa datos de ingreso de productos
3. if En "Estado" selecciona una de las siguientes opciones ("Dañado" / "No llego por pedido" / "Otro"):
    3.1. SYSTEM Convierte automaticamente el campo de "Descripcion" como obligatorio
    3.2. EMPRESARIOADMIN INVENTARIO ingresa una descripcion correctamente y guarda.
    3.3. SYSTEM valida los datos.
    3.4. SYSTEM registra el ingreso con la descripción.
    3.5. SYSTEM Muestra mensaje de éxito.
   end if
4.

Descripción vacía
Steps
1. if En paso 3.2, el SISTEMA detecta campo de descripción vacío:
    1.1. SYSTEM Muestra aviso indicando (Se requiere descripción detallada de los hechos por registro de producto en "Estado" )
    1.2. EMPRESARIOADMIN INVENTARIO acepta como recibida la informacion
    1.3. SYSTEM Le permite registrar una descripción nuevamente
   end if
