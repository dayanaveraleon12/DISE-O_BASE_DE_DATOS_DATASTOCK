Use Case Note
■ Workflow
    • // Write down briefly how user perform the work
    • El Administrador de Inventario o Empresario cuando requiere verificar la veracidad de un movimiento, realizar un arqueo de caja o prepararse para una auditoría externa.
    • El usuario accede al historial, filtra por fechas o productos, identifica el registro con soporte documental y visualiza el archivo adjunto para cotejarlo con la información física o contable.
    •
■ Business Logic
    • // Write down what user expect the system to react upon certain condition (e.g. low inventory alert level)
    • Integridad de Solo Lectura: El archivo consultado no puede ser editado ni reemplazado desde este módulo para evitar alteraciones de evidencia histórica.
    • Disponibilidad: El botón de consulta solo debe estar activo (o visible) si la transacción seleccionada posee un registro de archivo en la base de datos.
    • Seguridad de Acceso: Solo usuarios con rol de Administrador de Inventario o Empresario pueden visualizar respaldos; los empleados de nivel operativo tendrán esta opción restringida al igual que usuarios con rol "Pendiente".
■ Decisions
    • // Write down the decisions made during the meeting (e.g. Must allow accessing from mobile devices)
    • Ubicación Funcional: La consulta se integrará directamente en la tabla del módulo "Historial" mediante una columna adicional de "Acciones" o "Evidencia".
    • Previsualización Nativa: El sistema intentará abrir el archivo en una ventana modal interna (especialmente JPG, PNG o PDF) antes de forzar la descarga, para agilizar la revisión.
    • Manejo de Errores: Si por alguna razón técnica el archivo no se encuentra en el servidor/nube, el sistema debe mostrar un aviso: "Archivo no disponible o extraviado, contacte a soporte".
■ Follow-up
    • // Write down the items that should follow-up in the coming meeting
    • ¿Es necesario generar un log o registro de quiénes han consultado/descargado cada factura por razones de seguridad?
    • ¿Se debe permitir la descarga masiva de evidencias (ej. todas las facturas del mes o por ingreso masivo de productos) en un archivo .ZIP?

Scenario
Steps
1. EMPRESARIOADMIN INVENTARIO Selecciona la opción "Historial" en el Menú de Navegación de la plataforma. CONSULTAR HISTORIAL
2. SYSTEM Muestra la lista cronológica de todos los movimientos (Ingresos y Salidas).
3. EMPRESARIOADMIN INVENTARIO Utiliza el filtro de búsqueda (por rangos de fechas) para localizar la transacción deseada.
4. if EMPRESARIOADMIN INVENTARIO Selecciona en un registro de movimiento de su interes en "Ver detalle":El usuario identifica en la fila del registro un icono de "Ojo" o "Clip" que indica la existencia de un respaldo.
    4.1. SYSTEM Muestra una ventana modal con todo el detalle y para cada producto al finalizar la informacion de cada uno esta el titulo "Factura de proveedor" y al frente de este el icono de un "ojo" con la palabra "Ver"
    4.2. if EMPRESARIOADMIN INVENTARIO Desea ver la evidencia cargada en "Factura de proveedor":
        4.2.1. SYSTEM Carga una ventana modal que le permita la previsualización de esta y descargarla si lo desea
        4.2.2. if EMPRESARIOADMIN INVENTARIO Hace clic en el botón "Descargar".
            4.2.2.1. El sistema descarga el archivo al dispositivo local con su nombre original.
            4.2.2.2. EMPRESARIOADMIN INVENTARIO Cierra el modal y regresa a la tabla de historial.
               end if
         end if
   end if

Transacción sin evidencia
Steps
1. if EMPRESARIOADMIN INVENTARIO Visualiza que no puede ver el documento ni se habilita el boton de "ojo" para seleccionar esta opción:
    1.1. SYSTEM Genera un aviso informativo: "Esta transacción no cuenta con soporte documental".
    1.2. El flujo termina y el usuario continúa navegando en el historial.
   end if
