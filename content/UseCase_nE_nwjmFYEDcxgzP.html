Use Case Note
■ Workflow
    • // Write down briefly how user perform the work
    • El usuario (Empresario o Administrador) accede al formulario de "Registro de Producto".
    • Al no encontrar una opción requerida en las listas desplegables ( de Categoría, Marca o Presentación), selecciona el botón "Agregar Nuevo" / (+) junto al campo correspondiente.
    • El sistema despliega una ventana emergente (modal).
    • El usuario ingresa el nombre y descripción del nuevo parámetro.
    • El sistema valida que no exista un duplicado y guarda la información.
    • El nuevo dato aparece seleccionado automáticamente en el formulario principal.
■ Business Logic
    • // Write down what user expect the system to react upon certain condition (e.g. low inventory alert level)
    • Validación de Duplicados: El sistema debe verificar en tiempo real que el nombre de la Marca, Categoría o Presentación no exista previamente (ignorando mayúsculas/minúsculas) para evitar redundancia en los reportes de DATASTOCK.
    • Estado por Defecto: Toda nueva opción registrada se creará con estado "Disponible" de forma automática.
    • Integridad Referencial: No se permitirá eliminar una Marca o Categoría si ya tiene productos asociados en el inventario; solo se permitirá su desactivación (borrado lógico) cuando no tenga productos asociados.
■ Decisions
    • // Write down the decisions made during the meeting (e.g. Must allow accessing from mobile devices)
    • Acceso Rápido: Se decidió que la creación de estos parámetros no sea exclusiva de un módulo de "Configuración", sino que pueda hacerse "on-the-fly" (al vuelo) mientras se crea un producto para mejorar las opciones predeterminadas.
    • Auditoría: El sistema debe registrar qué usuario creó la nueva opción y en qué fecha, para mantener el control sobre el crecimiento del catálogo.
    • Estandarización: Los nombres ingresados se transformarán automáticamente a Proper Case (Primera letra mayúscula) para mantener la estética visual de la plataforma.
    • La nueva opción (Marca, Presentación o Categoría) queda disponible para futuros registros de productos en DataStock.
    • El formulario de "Agregar Productos" permanece abierto para que el usuario complete el resto de la información (Lote, Vencimiento, Factura, etc.) sin perder el progreso.
■ Follow-up
    • // Write down the items that should follow-up in the coming meeting
    • ¿Se requerirá subir un logotipo para cada Marca registrada o solo el nombre textual?
    • ¿Deben existir sub-categorías dentro del catálogo para una mejor organización de DATASTOCK?
    • Definir si el "Empresario" puede restringir a ciertos Administradores de Inventario el permiso de crear nuevas marcas para evitar desorden en la base de datos.

Scenario
Steps
1. EMPRESARIOADMIN INVENTARIO selecciona la opción "Inventario" en el Menú de Navegación principal.
2. SYSTEM muestra la vista de Stock actual y el botón "Agregar Inventario".
3. EMPRESARIOADMIN INVENTARIO hace clic en "Agregar Inventario".
4. SYSTEM despliega la ventana emergente "Agregar Productos al Inventario".
5. if EMPRESARIOADMIN INVENTARIO interactúa con los selectores de Marca, Categoría o Presentación y sí selecciona la opción (+) / "Agregar Nuevo" (ubicada al final de la lista predeterminada)
    5.1. SYSTEM despliega una ventana emergente (modal).
    5.2. SYSTEM habilita un campo de texto dinámico para escribir el nombre de la nueva opción.
    5.3. EMPRESARIOADMIN INVENTARIO ingresa el nombre (ej. "Nueva Marca X") y presiona el botón "Guardar".
    5.4. SYSTEM muestra un aviso de confirmación indicando el ítem (Marca/Presentación/Categoría) y el nombre ingresado.
    5.5. EMPRESARIOADMIN INVENTARIO selecciona "Continuar".
    5.6. SYSTEM registra la nueva opción en la base de datos, cierra el aviso y la selecciona automáticamente en el formulario principal.
   end if

El usuario cancela la creación
Steps
1. if EMPRESARIOADMIN INVENTARIO visualiza el aviso de confirmación con el nombre de la nueva opción.
    1.1. EMPRESARIOADMIN INVENTARIO selecciona "Cancelar".
    1.2. SYSTEM descarta los cambios, cierra el aviso y devuelve al usuario al selector original sin realizar cambios en la base de datos.
   end if

Opción Duplicada
Steps
1. if EMPRESARIOADMIN INVENTARIO ingresa un nombre que ya existe en los registros actuales (ej. "Alpina").
    1.1. SYSTEM detecta la coincidencia y muestra un mensaje de error: "La opción ingresada ya existe en el sistema".
    1.2. SYSTEM resalta el campo en rojo y permite al usuario corregir el nombre o seleccionar la opción existente de la lista.
   end if
