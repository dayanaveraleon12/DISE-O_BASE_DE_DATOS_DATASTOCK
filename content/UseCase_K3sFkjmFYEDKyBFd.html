Use Case Note
■ Workflow
    • // Write down briefly how user perform the work
    • El Administrador de Inventario ó empresario inicia sesión.
    • El actor va a encontrar un botón que diga "Inventario".
    • Selecciona la opción “Ingresar Stock / Agregar inventario”.
    • Ingresa:
        ♦ Nombre del Producto
        ♦ Marca
        ♦ Categoría (predeterminadas)
        ♦ Presentación (predeterminadas)
        ♦ Ubicación (Bodega, Tienda #)
        ♦ Cantidad recibida
        ♦ Número de lote
        ♦ Fecha de ingreso
        ♦ Fecha de vencimiento
        ♦ Estado: o Normal / o Dañado / o No llego por pedido / o Otro
        ♦ Descripción (por sí algún producto llega dañado o algo similar)
        ♦ Sube factura de Proveedor para tener respaldo y validez
    • El actor confirma el registro y selecciona "Guardar productos".
    • El sistema valida los datos.
    • El sistema actualiza la cantidad disponible en bodega.
    • El sistema guarda el movimiento en el historial.
    • Cuando ingresa en ésta opción, el sistema va a redirigir al actor a la pantalla de ésta función.
■ Business Logic
    • // Write down what user expect the system to react upon certain condition (e.g. low inventory alert level)
    • No se puede ingresar una cantidad negativa.
    • La fecha de vencimiento no puede ser anterior a la fecha actual / de ingreso.
    • Cada ingreso debe generar un registro en el historial de movimientos.
    • El sistema debe actualizar automáticamente el total disponible en bodega o en el numero de tienda seleccionado y en el stock general de todo según cantidad ingresada.
■ Decisions
    • // Write down the decisions made during the meeting (e.g. Must allow accessing from mobile devices)
    • Si los datos están incompletos → no permite guardar.
    • Si la validación es correcta → actualiza el stock.
    • Si ocurre un error en base de datos → muestra aviso de falla de registro.
■ Follow-up
    • // Write down the items that should follow-up in the coming meeting
    • N/A

Scenario
Steps
1. EMPRESARIOADMIN INVENTARIO selecciona la opción “Ingresar Stock”.
2. SYSTEM Muestra el formulario de ingreso de productos
3. EMPRESARIOADMIN INVENTARIO ingresa datos obligatorios en campos individuales:
    3.1. Nombre del Producto
    3.2. Marca
    3.3. Categoría (predeterminadas)
    3.4. Presentación (predeterminadas)
    3.5. Ubicación (Bodega, Tienda #)
    3.6. Cantidad recibida
    3.7. Número de lote
    3.8. Fecha de ingreso
    3.9. Fecha de vencimiento
    3.10. Estado: o Normal / o Dañado / o No llego por pedido / o Otro
    3.11. Descripción (por sí algún producto llega dañado o algo similar)
    3.12. Sube factura de Proveedor para tener respaldo y validez
4. El EMPRESARIOADMIN INVENTARIO confirma el registro.
5. SYSTEM Valida:
    5.1. Que la fecha de vencimiento sea válida.
    5.2. Que la cantidad sea mayor a cero.
    5.3. Que el lote no esté duplicado a los previos registrados.
6. SYSTEM Guarda el lote en base de datos.
7. SYSTEM Actualiza el stock total del producto.
8. SYSTEM Registra el movimiento en el historial.
9. SYSTEM Muestra mensaje: “Stock ingresado correctamente”.
