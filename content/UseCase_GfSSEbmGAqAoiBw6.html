Use Case Note
■ Workflow
    • // Write down briefly how user perform the work
    • El Empresario accede al módulo de "Gestión de Evidencias" y selecciona la opción para cargar un nuevo archivo.
    • El usuario arrastra o selecciona el archivo Excel (.xlsx/.csv) generado por su sistema de ventas diario y selecciona la tienda de origen.
    • El sistema muestra una previsualización de los datos cargados (ocultando valores financieros) para que el usuario confirme el envío.
■ Business Logic
    • // Write down what user expect the system to react upon certain condition (e.g. low inventory alert level)
    • Validación de Formato: El sistema debe restringir la carga a extensiones Excel; de lo contrario, debe disparar un mensaje de error "Formato no compatible".
    • Filtro de Privacidad (Data Stripping): Al procesar el archivo, el sistema debe eliminar automáticamente cualquier columna relacionada con "Precios", "Costos" o "Utilidades" antes de guardar la información en la base de datos de inventario.
    • Mapeo de Columnas: El sistema debe identificar por medio de palabras clave (Keywords) las columnas obligatorias: Producto, Presentación, Marca y Cantidad. Si falta una, el registro no se completa.
    • Disponibilidad para Admin: Una vez confirmada la subida, el Admin de Inventario debe ver una notificación en su panel de "Salida de Productos" para proceder con el descuento de stock físico.
■ Decisions
    • // Write down the decisions made during the meeting (e.g. Must allow accessing from mobile devices)
    • Almacenamiento Temporal: Se decidió que el archivo original no se almacena íntegramente; solo se extraen y guardan los datos filtrados para optimizar el espacio en el servidor.
    • Acceso Multi-Plataforma: El Empresario debe poder realizar la subida desde dispositivos móviles, facilitando el envío de evidencias directamente desde el punto de venta.
    • Unicidad de Carga: No se permitirá subir dos archivos para la misma tienda en el mismo día calendario sin una autorización previa para evitar duplicidad de salidas.
■ Follow-up
    • // Write down the items that should follow-up in the coming meeting
    • Determinar qué sucede si el nombre de un producto en el Excel no coincide exactamente con el nombre en la base de datos de DataStock (Sincronización de data).

Scenario
Steps
1. EMPRESARIO selecciona la opción "Subir Evidencia de Ventas" en el panel de control.
2. SYSTEM solicita la carga del archivo y la selección de la tienda/sucursal correspondiente.
3. EMPRESARIO adjunta el archivo Excel y confirma el envío.
4. SYSTEM valida que el formato sea correcto y que el archivo no esté corrupto.
5. SYSTEM aplica el filtro de privacidad, extrayendo únicamente las columnas: Producto, Presentación, Marca y Cantidad.
6. SYSTEM almacena el registro y genera una notificación automática para el Admin. Inventario.
7. SYSTEM confirma al Empresario que el archivo se subió con éxito.
