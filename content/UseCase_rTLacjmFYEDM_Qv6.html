Use Case Note
■ Workflow
    • // Write down briefly how user perform the work
    • En la interfaz de iniciar sesion, el usuario encuentra la opcion “Olvide contraseña”.
    • Debe llenar el campo de correo electrónico y da click en el botón.
    • Se genera un aviso de correo enviado de recuperación de contraseña
    • Ingresa los 6 digitos de seguridad que se enviarón al correo en el campo de dígitos.
    • Si coninciden los digitos de seguridad, le permite cambiar la contraseña. De lo contrario, no puede cambiar la contraseña. y se le envía un nuevo código para intentar cambiarla nuevamente.
■ Business Logic
    • // Write down what user expect the system to react upon certain condition (e.g. low inventory alert level)
    • El sistema valida que se hayan escrito más de 8 caracteres en el campo donde se solicita el correo.
    • Hay dos campos donde el usuario va a poder colocar la nueva contraseña y confirmarla. Éstos campos deben coincidir para cambiarla correctamente.
■ Decisions
    • // Write down the decisions made during the meeting (e.g. Must allow accessing from mobile devices)
    • Se decidió que la recuperación de contraseña es un proceso autónomo para cada usuario.
■ Follow-up
    • // Write down the items that should follow-up in the coming meeting
    • N/A

Scenario
Steps
1. 1. Al intentar iniciar sesion, cualquier actor puede tener una alerta de "Contraseña incorrecta", y si da click en "Olvide contraseña", puede recuperarla.
2. EMPRESARIOADMIN INVENTARIOEMPLEADO selecciona la opción "Olvidé contraseña" en el Login.
3. EMPRESARIOADMIN INVENTARIOEMPLEADO llena el campo con su correo:
    3.1. SYSTEM Valida que se hayan escrito más de 8 caracteres.
    3.2. SYSTEM Busca en la base de datos que el correo esté registrado.
    3.3. if El correo está regitsrado en la base de datos, el sistema envía un código de 6 dígitos para colocarlo en la plataforma y así cambiar la contraseña.
        3.3.1. if Si coinciden los 6 digitos de seguridad ingresados, el usuario procede a escribir la nueva contraseña y confirmarla.
            3.3.1.1. SYSTEM Activa el botón "Cambiar Contraseña".
            3.3.1.2. EMPRESARIOADMIN INVENTARIOEMPLEADO hace clic en "Cambiar contraseña".
            3.3.1.3. SYSTEM Guarda el cambio de contraseña en la base de datos.
               end if
         end if
    3.4. if Si el correo no está registrado, el sistema arroja un mensaje de alerta indicando que en el correo no está en la base de datos.
         end if
