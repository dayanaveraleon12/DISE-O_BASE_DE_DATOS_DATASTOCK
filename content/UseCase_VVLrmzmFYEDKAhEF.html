Use Case Note
■ Workflow
    • // Write down briefly how user perform the work
    • El usuario (Admin, inventario y Empresario) ingresa a la opcion "inventario" y luego a la opcion "agregar inventario".
    • Tras completar los datos del ingreso, el usuario hace "clic" en el boton adjuntar respaldo y selecciona el archivo desde su dispositivo.
    • El sistema sube el archivo temporalmente y verifica que cumpla con los formatos y pesos permitidos.
    • Al confirmar la operacion el sistema guarda el archivo de forma definitiva en el servidor y lo enlaza directamente al ID del movimiento registrado en la base de datos.
■ Business Logic
    • // Write down what user expect the system to react upon certain condition (e.g. low inventory alert level)
    • El sistema debe aceptar extensiones de documentos e imagenes estandar, como PDF, JPG, o PNG.
    • Se debe establecer un tamaño maximo por archivo (5 MB O 10 MB) para optimizar el espacio de almacenamiento en la base de datos o servidor en la nube.
    • Un archivo de respaldo no puede existir "flotando" en el sistema, debe estar ligado obligatoriamente a una operacion de inventario especifica.
■ Decisions
    • // Write down the decisions made during the meeting (e.g. Must allow accessing from mobile devices)
    • El sistema decide si aprueba o rechaza el documento basandose en su extension. Si es un archivo no admitido, muestra un error.
    • Evalua si el peso del archivo sobrepasa el limite. Si es asi, solicita al usuario que comprima la imagen o el PDF.
    • Si el (Admin Inventario o Empresario) al ingresar la informacion no sube el respaldo de imagen, no debe permitir seguir el proceso y viceversa.
    • Decide mostrar un indicador visual (como un check verde) para confirmar que el documento esta listo para subirse.
    • El movimiento de inventario queda registrado con su respectiva evidencia documental.
    • El archivo queda disponible en el módulo de "Historial" mediante un icono de visualización para futuras auditorías o cruces de cuentas.
■ Follow-up
    • // Write down the items that should follow-up in the coming meeting
    • El Empresario o el Admin de Inventario podrá abrir, visualizar y descargar este archivo en cualquier momento para auditorias o cruces de cuentas.
    • Implementar un compresor de archivos pdf y imágenes con el fin de no tener que acudir a externos para comprimir las imágenes o pdf.

Scenario
Steps
1. El EMPRESARIOADMIN INVENTARIO se encuentra en la ventana "Agregar Productos al Inventario" completando los datos de una nueva entrada.
2. Se desplazan hasta la sección "Factura del Proveedor (JPG)".
3. Le dan "clic" en el botón de carga (ícono de subir archivo) o selecciona el área de adjunto.
4. SYSTEM Abre el explorador de archivos del dispositivo.
5. El EMPRESARIOADMIN INVENTARIO selecciona el archivo digital (PDF, JPG o PNG) que contiene el respaldo de la factura.
6. SYSTEM Valida que el archivo no supere el límite de tamaño establecido (ej. 5 MB) y que la extensión sea permitida.
7. SYSTEM Muestra un indicador visual (check verde) confirmando que el documento es válido y está listo para procesarse.
8. El EMPRESARIOADMIN INVENTARIO completa el resto del formulario y presiona "Guardar Productos".
9. SYSTEM Procesa la transacción de inventario y vincula de forma permanente el archivo al registro del movimiento.
10. SYSTEM Confirma el éxito de la operación y limpia el almacenamiento temporal.

Archivo con extensión no permitida
Steps
1. El sistema detecta que el archivo no es PDF, JPG o PNG.
2. El sistema muestra un mensaje de error: "Formato de archivo no admitido. Por favor, suba una imagen o un PDF".
3. El sistema bloquea el indicador de confirmación y permite al usuario seleccionar un nuevo archivo.

Archivo excede el tamaño máximo
Steps
1. El sistema detecta que el archivo supera el límite de peso (ej. > 5 MB).
2. El sistema muestra un aviso: "El archivo es demasiado pesado. Por favor, comprima la imagen o el PDF antes de subirlo".
3. El usuario debe optimizar el archivo y repetir el proceso de carga.

Cancelación del ingreso de mercancía
Steps
1. El usuario, tras haber cargado el archivo, decide presionar el botón "Cancelar" en el formulario principal de registro.
2. El sistema descarta los datos del formulario y elimina automáticamente el archivo temporal del servidor para evitar la acumulación de "basura" digital.
3. El sistema regresa a la vista de Inventario General sin generar ningún registro.
