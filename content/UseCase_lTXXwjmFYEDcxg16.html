Use Case Note
■ Workflow
    • // Write down briefly how user perform the work
    • Inician sesión con el usuario y contraseña enviado a su correo personal registrado, y que también fueron almacenados en base de datos
    • Si coinciden con los registrados en el sistema, lo remite a la ventana correspondiente.
    • De lo contrario le indica “Usuario o contraseña inválidos”
■ Business Logic
    • // Write down what user expect the system to react upon certain condition (e.g. low inventory alert level)
    • Si los campos de ingreso de sesión están completos y no falta alguno por diligenciar, se habilitará el botón para hacer el Login
    • El sistema procesa y valida que el usuario y contraseña ingresados coinciden con los registrados previamente en bases de datos
    • Si los datos ingresados coinciden con los registrados en el sistema, se genera un aviso de “Iniciaste sesión correctamente”. Si no, indica “Usuario o contraseña inválidos”
■ Decisions
    • // Write down the decisions made during the meeting (e.g. Must allow accessing from mobile devices)
    • Para poder validar la contraseña, el sistema debe tener en cuenta también el usuario a buscar
■ Follow-up
    • // Write down the items that should follow-up in the coming meeting
    • N/A

Scenario
Steps
1. SYSTEM Valida la base de datos y verifica que esté registrado el usuario.
2. if Si está registrado, valida la contraseña ingresada con la almacenada.
    2.1. if Si es correcta, permite el ingreso del usuario en cuestión.
        2.1.1. if Si no es correcta, el sistema arroja un mensaje de alerta indicando que alguno de los datos ingresados es incorrecto.
               end if
         end if
   end if

Contraseña Incorrecta
Steps
1. if Sí la contraseña es incorrecta:
    1.1. SYSTEM genera un aviso indicando "Contraseña incorrecta" y recomienda el cambio de esta CAMBIAR CONTRASEÑA
    1.2. SYSTEM Limpia el campo de entrada de "Contraseña" y vuelve al escenario exitoso inicial permitiendo el ingreso de datos nuevamente
   end if

Preconditions El usuario debe haber ingresado sus credenciales (correo/usuario y contraseña) en la interfaz del sistema, y la plataforma debe tener conexión activa con la base de datos.
Post-conditions El sistema confirma que la contraseña ingresada coincide con el registro (validando el cifrado), aprueba la identidad del usuario y permite continuar con el proceso de acceso.
