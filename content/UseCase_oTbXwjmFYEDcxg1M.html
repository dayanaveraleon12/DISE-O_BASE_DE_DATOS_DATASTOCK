Use Case Note
■ Workflow
    • // Write down briefly how user perform the work
    • Cuando el actor inicia sesión por medio del Login de acuerdo al ROL con el que fue registrado, se va a mostrar su ventana correspondiente y va a poder modificar cierta información o no.
■ Business Logic
    • // Write down what user expect the system to react upon certain condition (e.g. low inventory alert level)
    • Cuando el usuario inicia sesión en el sistema, se valida el rol asignado al usuario en cuestión:
    • Empresario:
        ♦ Puede modificar más datos que los otros actores o los datos de estos.
    • Admin de inventario:
        ♦ Él puede editar únicamente lo que se trata de inventario, listas y el stock.
    • Empleado:
        ♦ Puede visualizar la información de inventario y stock, además de enviar listas con solicitudes de productos.
    • Pendiente:
        ♦ Solo puede ver una página en blanco.
■ Decisions
    • // Write down the decisions made during the meeting (e.g. Must allow accessing from mobile devices)
    • De acuerdo con el rol asignado cada actor puede ver una información específica o determinadas acciones.
■ Follow-up
    • // Write down the items that should follow-up in the coming meeting
    • N/A

Scenario
Steps
1. SYSTEM Recupera de la base de datos la etiqueta de rol asignada al usuario (EMPRESARIO, ADMIN INVENTARIOEMPLEADO, y/o Pendiente).
2. Se activa la interfaz personalizada según el nivel de autorización.
3. if Si el rol es ADMIN INVENTARIO:
    3.1. Se activan los botones de registro y modificación de productos.
    3.2. if Si el rol es EMPLEADO:
        3.2.1. el sistema bloquea los formularios de edición y solo permite consultas de productos de inventario y creación de remisiones.
        3.2.2. if Si el rol es EMPRESARIO:
            3.2.2.1. tiene la facultad de monitorear registros y movimientos por parte del ADMIN INVENTARIO y eliminar usuarios del sistema por medio de cambio de rol a "Pendiente", asi como subir las evidencias para el registro de salida de productos
            3.2.2.2. if Sí por el contrario no tiene un rol asignado sino fue registrado con la opición en estado "Pendiente":
                3.2.2.2.1. Tendra acceso solo a una pantalla vacia con un mensaje indicando "Solicite acceso a su jefe inmediato", despues de 7 meses en este mismo estado el usuario será eliminado
                     end if
               end if
         end if
   end if
4. El EMPRESARIO ADMIN INVENTARIO está autorizado para crear categorías de productos, registrar ingreso y/o salida de productos.
5. Todos los roles registrados tienen permiso para visualizar la cantidad total de productos en stock e inventario. CONSULTAR INVENTARIO
