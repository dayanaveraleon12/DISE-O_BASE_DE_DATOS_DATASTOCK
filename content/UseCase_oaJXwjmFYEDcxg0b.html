Use Case Note
■ Workflow
    • // Write down briefly how user perform the work
    • El Administrador de Inventario ó Empresario "Inicia sesión", se dirige a "Inventario" y "Agregar inventario"
    • Dentro de esa ventana modal: en la opción “Número Lote”.
    • El actor ingresa:
        ♦ Número de lote
        ♦ Fecha de vencimiento
    • El administrador confirma el registro.
    • El sistema valida los datos.
    • El sistema guarda el lote asociado al producto.
    • El sistema activa la verificación automática de vencimiento.
    • El sistema muestra mensaje de confirmación.
■ Business Logic
    • // Write down what user expect the system to react upon certain condition (e.g. low inventory alert level)
    • El número de lote no puede repetirse con productos previos ya registrados con diferente fecha de vencimiento.
    • La fecha de vencimiento no puede ser anterior a la fecha actual.
    • Cada lote debe estar vinculado a un ó unos productos.
    • El sistema debe verificar diariamente los lotes.
    • Si faltan 7 días ó menos para vencer → generar alerta automática.
    • Si el lote está vencido → marcar como “Vencido” y notificar.
■ Decisions
    • // Write down the decisions made during the meeting (e.g. Must allow accessing from mobile devices)
    • Solo se van a permitir ingresar productos con el mismo lote de productos previos registrados sí estos tienen el mismo lote y la misma fecha de caducidad (teniendo en cuenta que el proveedor no haya agotado ese lote)
    • Si el número de lote ya existe y se va a registrar con una fecha distinta al registro existente → no permitir registro.
    • Si la fecha es inválida → no permitir guardar.
    • Si la validación es correcta → registrar lote vinculado al/los productos.
    • Si faltan 7 días para vencer → enviar alerta al Administrador y Empresario.
■ Follow-up
    • // Write down the items that should follow-up in the coming meeting
    • Generar reporte mensual de vencimientos.
    • Implementar método PEPS (primero en entrar primero en salir) automático para salida de productos, por ejemplo: dañados.

Scenario
Steps
1. EMPRESARIOADMIN INVENTARIO ingresa a CONSULTAR INVENTARIO
2. ADMIN INVENTARIOEMPRESARIO selecciona la opción "Agregar inventario"
3. EMPRESARIOADMIN INVENTARIO dentro de la ventana modal de INGRESOS DE PRODUCTOS Ingresa datos por teclado:
    3.1. En la opción “Número de lote” y "Fecha de vencimiento".
    3.2. SYSTEM Permite ingresar las Fechas de los Lotes en formato DD/MM/AAAA en la sección "Fecha de vencimiento" y "Fecha de Ingreso"
    3.3. SYSTEM Solo va a permitir asignar un lote por producto.
    3.4. SYSTEM El lote no debe coincidir con previos ya registrados
4. ADMIN INVENTARIOEMPRESARIO Llena los demás campos de ingreso de productos
5. ADMIN INVENTARIOEMPRESARIO Guarda la información de ingreso de productos.
6. SYSTEM Guarda el lote en la base de datos y posterior sigue con el caso de uso SISTEMA NOTIFICAR CADUCIDAD PRODUCTOSy finaliza el de CONSULTAR INVENTARIO
