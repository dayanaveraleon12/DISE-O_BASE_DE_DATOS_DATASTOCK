Name Description
 LOG IN : Use Case

EL SISTEMA SOLICITA LA VALIDACION DE LOS DATOS (CORREO ELETRÓNICO - CONTRASEÑA) PARA CONSULTAR EL PERFIL DEL USUARIO EN LA BASE DE DATOS.

 RECIBIR NOTIFICACIONES INGRESO : Use Case

SE ENVÍA UNA NOTIFICACIÓN AUTOMÁTICA AL EMPRESARIO VÍA CORREO ELECTRÓNICO CUANDO INGRESA EL ADMIN DE INVENTARIO EN LA PLATAFORMA WEB CON LA HORA DE INICIO DE SESION.

 CREAR USUARIOS : Use Case

A PARTIR DE REGISTROS DE DATOS BÁSICOS COMO NOMBRE, CORREO, CONTRASEÑA, Y SELECCION DE ROL CREAR LOS USUARIOS AUTORIZADOS PARA QUE INTERACTUEN CON LA PLATAFORMA WEB. SOLO AUTORIZADO POR EL EMPRESARIO.

 CREAR MARCA, CATEGORIA Y PRESENTACION : Use Case

PERMITE REGISTRAR UNA NUEVA OPCION INDIVIDUALMENTE PARA OPCIONES DE INGRESO COMO CATEGORÍA, MARCA Y PRESENTACION EN LA PLATAFORMA DE SER NECESARIO.

 CREAR LOTES - PRODUCTOS : Use Case

EL ADMIN DEL INVENTARIO Ó EMPRESARIO PUEDE CREAR UN LOTE (SIENDO UNICOS POR INGRESO DE PRODUCTOS) O ASOCIARLO JUNTO CON UNA FECHA DE CADUDIDAD A UN PRODUCTO YA EXISTENTE (POR EJEMPLO PARA LA BUSQUEDA DE INVENTARIO POR MEDIO DE FILTROS)

 VALIDAR ROLES : Use Case

EL SISTEMA BUSCA EN LA BASE DE DATOS LOS PERMISOS Y ACCESOS OTORGADOS AL USUARIO DE ACUERDO A SU ROL ASIGNADO EN EL MOMENTO DE SU REGISTRO Ó MODIFICACION

 VALIDAR CONTRASEÑA : Use Case

 EL SISTEMA VA A CONTRASTAR LAS CREDENCIALES INGRESADAS CON LA BASE DE DATOS PARA PERMITIR EL INGRESO AL SISTEMA; SI LOS DATOS COINCIDEN SE LE DA ACCESO AL ENTORNO DE TRABAJO; DE LO CONTRARIO, SE LE DA UN MENSAJE DE ALERTA DE CONTRASEÑA INVALIDA Y RECOMENDACION PARA QUE CAMBIE LA CONTRASEÑA.

 SISTEMA MOSTRAR ESTADÍSTICAS : Use Case

PERMITE AL EMPRESARIO, AL ADMIN. DE INVENTARIO Y EMPLEADO, CONSULTAR INFORMACION DE LOS 5 ARTICULOS MÁS VENDIDOS (CON MAS SALIDAS DEL INVENTARIO CON MOTIVO DE SALIDA "VENTA") DEL STOCK EN EL MES Y LOS 5 MENOS VENDIDOS (PRODUCTOS CON MENOS SALIDAS DEL INVENTARIO CON MOTIVO DE SALIDA "VENTA"), CON MOTIVO DE SALIDA DE "VENTA" PARA NO TENER PRESENTE AQUELLOS QUE SALIERON POR DAÑO, HURTO, CONSUMO INTERNO U OTROS, SINO LOS SOLICITADOS POR CLIENTES EN COMPRAS.

 SISTEMA NOTIFICAR CADUCIDAD PRODUCTOS : Use Case

CUANDO EL SISTEMA DETECTE QUE DESDE LA FECHA DE INGRESO ALGUNOS DE LOS ARTÍCULOS SE ACERCAN A SU CADUCIDAD, SE GENERARÁ UNA NOTIFICACIÓN ANTICIPADA AUTOMÁTICA (1 MES, 15 DÍAS Y EL MISMO DÍA) AL EMPRESARIO, ADMIN. DE INVENTARIO Y EMPLEADO DENTRO DE LA MISMA PLATAFORMA PARA QUE PUEDAN PRIORIZAR LOS MISMOS Y TOMAR MEDIDAS.

 ASIGNAR ROLES : Use Case

DE ACUERDO AL CARGO QUE VA A TENER EL USUARIO, SE LE DARÁ UN CONJUNTO ESPECÍFICO DE PERMISOS OPERATIVOS DENTRO DE LA PLATAFORMA WEB.

 MODIFICAR : Use Case
 CONSULTAR INVENTARIO : Use Case

CUALQUIER USUARIO PUEDE VISUALIZAR LA CANTIDAD DE ARTICULOS DISPONIBLE QUE SE ENCUENTRAN EN LA BODEGA, LAS TIENDAS, O EL INVENTARIO EN GENERAL.

 MODIFICAR DATOS PERSONALES : Use Case

EL EMPRESARIO VA A TENER HABILITADA ÉSTA FUNCIÓN PARA ACTUALIZAR LA INFORMACIÓN EN LOS CAMPOS EDITABLES COMO: NOMBRES, APELLIDOS, CORREO, EPS, CONTACTO DE EMERGENCIA, CELULAR. TRAS REALIZAR LOS CAMBIOS, EL SISTEMA VALIDA QUE LOS NUEVOS DATOS CUMPLAN CON LOS FORMATOS REQUERIDOS.

 MODIFICAR ROLES : Use Case

SELECCIONAR UN USUARIO CON UN ROL ASIGNADO, DONDE SE PUEDE SUSTITUIR EL ROL ACTUAL POR UNO NUEVO. TRAS LA MODIFICACION Y VALIDACION, EL SISTEMA ACTUALIZA INMEDIATAMENTE LOS NIVELES DE ACCESO AL USUARIO.

 

TENIENDO PRESENTE LOS ROLES:

- EMPRESARIO

- ADMINISTRADOR DE INVENTARIO

- EMPLEADO

- PENDIENTE (VISUALIZA UNA PANTALLA EN BLANCO SI SU ROL NO SE MODIFICA DESPUES DE 7 MESES SE ELIMINA EL USUARIO CREADO)

USUARIO
 CONSULTAR EVIDENCIAS DE INGRESOS Y SALIDAS : Use Case

PERMITE AL ADMINISTRADOR O EMPRESARIO LOCALIZAR, PREVISUALIZAR Y DESCARGAR LOS ARCHIVOS DIGITALES (FACTURAS O ACTAS) QUE FUERON VINCULADOS A UNA TRANSACCIÓN ESPECÍFICA DE ENTRADA O SALIDA DE MERCANCÍA EN EL HISTORIAL DEL SISTEMA.

 INGRESOS DE PRODUCTOS : Use Case

PERMITE AL ADMIN DEL INVENTARIO Ó EMPRESARIO INGRESAR UNA CANTIDAD DE ARTICULOS YA EXISTENTES EN EL INVENTARIO O NUEVOS, LLENANDO INFORMACION PARA CADA PRODUCTO A INGRESAR COMO: NOMBRE, MARCA, CATEGORIA, PRESENTACION, UBICACION, LA CANTIDAD A INGRESAR, #LOTE FECHA DE INGRESO, FECHA VENCIMIENTO Y EL SISTEMA ACTUALIZA LA CANTIDAD DISPONIBLE DE CADA PRODUCTO CON TODA SU INFOMACION

 CAMBIAR CONTRASEÑA : Use Case

EN CASO DE QUE EL ACTOR OLVIDE SU CONTRASEÑA, PUEDE CAMBIARLA AUTONOMAMENTE DESDE LA OPCIÓN OLVIDÉ MI CONTRASEÑA DESDE LOGIN.

 CONSULTAR HISTORIAL : Use Case

PERMITE AL ADMINISTRADOR DE INVENTARIO Y AL EMPRESARIO VISUALIZAR EL HISTORIAL DE MODIFICACIONES REALIZADAS EN EL SISTEMA, INCLUYENDO ACCIONES COMO ENTRADAS Y SALIDAS DE PRODUCTOS, JUNTO CON INFORMACIÓN DETALLADA DE CADA EVENTO (USUARIO, FECHA, HORA Y CAMBIOS REALIZADOS).

 CREAR REMISION : Use Case

EL EMPLEADO CREA UNA LISTA EN LA PLATAFORMA CON LOS ARTÍCULOS FALTANTES EN LA TIENDA, PARA QUE LE LLEGUE AL ADMIN. DE INVENTARIO O AL EMPLEADO DE OTRA TIENDA, Y ÉSTE PUEDA PREPARAR LA MERCANCÍA PARA LA TIENDA SOLICITANTE.

 CONSULTAR HISTORIAL DE REMISIONES : Use Case

TODOS LOS USUARIOS VAN A VER LAS LISTAS GESTIONADAS O PENDIENTES CON LOS ARTÍCULOS ENTREGADOS O RECHAZADOS POR NOVEDADES EN EL INVENTARIO DE ACUERDO A SI FUE SOLICITADA AL ADMINISTRADOR DE INVENTARIO O A UN EMPLEADO DE OTRA TIENDA.

 DESCRIPCION DE INGRESO : Use Case

EL EMPRESARIO Ó EL ADMINISTRADOR DE INVENTARIO PUEDEN AÑADIR A UN PRODUCTO EN PROCESO DE INGRESO, UNA DESCRIPCION PARA INDICAR EL ESTADO DE ESTE O LO OCURRIDO CON ESTE U CUALQUIER OTRA OBSERVACION

 
 CARGAR RESPALDO DE ENTRADA : Use Case

PUEDE CARGAR LA IMAGEN O PDF DE LA FACTURA DE PROVEEDOR BRINDADA AL ADMINISTRADOR DE INVENTARIO AL CREAR UN INGRESO DE PRODUCTOS

 SALIDA DE PRODUCTOS : Use Case

EL ADMINISTRADOR DE INVENTARIO Y EMPRESARIO, PUEDEN REGISTRAR LA SALIDA DE PRODUCTOS, LLENANDO LOS CAMPOS SOLICITADOS Y SUBIENDO UNA EVIDENCIA VALIDA COMO EL EXCEL DEL TOTAL DE VENTAS QUE LE COMPARTIERON A ÉL: LOS EMPLEADOS DE CADA TIENDA, EL SISTEMA LE PERMITE SUBIR ESTE ARCHIVO, LEE LOS PRODUCTOS Y CANTIDAD DE CADA UNO, UNA VEZ LOS LEE, EL SISTEMA LE MUESTRA LOS PRODUCTOS A REGISTRAR LA SALIDA; POSTERIOR A ESTO EL USUARIO VALIDA QUE SEA CORRECTO EL ANALISIS O LO CORRIGE MANUALMENTE Y GUARDA EL REGISTRO.

 LOG OUT : Use Case

LOS USUARIOS VAN FINALIZAR SU SESIÓN EN LA PLATAFORMA, PARA LUEGO REDIRIGIRLOS A LA PANTALLA DE INICIO DE SESIÓN.

 CARGAR Y PROCESAR RESPALDO DE SALIDA0 : Use Case

POR MEDIO DE ESTA OPCION QUE SE ENCUENTRA DENTRO DEL FORMULARIO / VENTANA DE "REGISTRO DE SALIDA" CON DEMAS OPCIONES DE ENTRADA DE DATOS, EL ADMINISTRADOR DE INVENTARIO O EMPRESARIO, CARGA EL EXCEL COMPARTIDO POR EMPLEADO SEGUN TIENDA ASIGNADA, SIENDO LA EVIDENCIA DE RESPALDO PARA EDITAR Y DESCONTAR PRODUCTOS EN INVENTARIO

 DESCRIPCION DE SALIDA : Use Case

EL APARTADO DE DESCRIPCION DE SALIDA ES OBLIGATORIA SOLO SI SELECCIONA UN MOTIVO DE SALIDA DE INVENTARIO COMO (Daño, Consumo interno, Vencimiento), PARA OTROS MOTIVOS DIFERENTES ES OPCIONAL Y ESTE APARTADO, PARA INGRESAR UNA DESCRIPCION APARACE AL CASI FINALIZAR EL PROCESO DE REGISTRO DE SALIDA DE INVENTARIO

 SUBIR Y DESCARGAR EVIDENCIA : Use Case

EL EMPRESARIO POR MEDIO DE ESTA OPCIÓN PUEDE SUBIR CONFIDENCIALMENTE EL ARCHIVO TIPO EXCEL (QUE ES EL DESCARGADO DE SU SISTEMA DE VENTAS CON LOS PRODUCTOS VENDIDOS DEL DIA POR TIENDA) PARA QUE EL ADMIN DE INVENTARIO QUIEN REGISTRARA USUALMENTE LA SALIDA DE LOS  PRODUCTOS, PUEDA DESCARGAR ESTE ARCHIVO, PERO EL SISTEMA PROCESARA SOLO LAS COLUMNAS REQUERIDAS COMO: PRODUCTO, PRESENTACION, MARCA Y CANTIDAD, (PARA MANTENER PRIVACIDAD EN CUANTO A GANANCIAS U PRECIOS TOTALES).
