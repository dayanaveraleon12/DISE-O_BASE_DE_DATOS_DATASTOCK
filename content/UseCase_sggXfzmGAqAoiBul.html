Use Case Note
■ Workflow
    • // Write down briefly how user perform the work
    • El Administrador de inventario o Empresario, durante el registro de salida de productos, selecciona un archivo de respaldo (Excel) desde su dispositivo.
    • El sistema valida el archivo, lo carga y lo procesa para extraer los productos y cantidades.
    • Posteriormente, muestra la información al usuario para su validación antes de continuar con el registro de salida.
■ Business Logic
    • // Write down what user expect the system to react upon certain condition (e.g. low inventory alert level)
    • Solo permite archivos en formato Excel (.xls, .xlsx)
    • Valida la estructura del archivo (columnas requeridas)
    • Procesa el contenido para extraer productos y cantidades
    • Verifica que los productos existan en el inventario
    • Genera una vista previa editable
    • Asocia el archivo como evidencia al registro de salida
    • Si el archivo presenta errores:
        ♦ Notifica inconsistencias (formato, datos faltantes, productos no encontrados) y permite subir un archivo de nuevo
    • Si todo esta bien: permite al usuario corregir manualmente antes de continuar y registrar
■ Decisions
    • // Write down the decisions made during the meeting (e.g. Must allow accessing from mobile devices)
    • El archivo Excel es obligatorio como evidencia de salida
    • Debe existir un formato estándar definido (Producto con descripcion de nombre y medida: kl, lb, unidad, etc / marca / presentacion / cantidad)
    • El sistema debe permitir correcciones manuales posteriores a la carga
    • Tanto el administrador como el empresario pueden cargar el archivo
    • El archivo debe quedar almacenado para auditoría posterior
■ Follow-up
    • // Write down the items that should follow-up in the coming meeting
    • Definir plantilla oficial del Excel
    • Establecer reglas de validación de columnas
    • Evaluar límite de tamaño del archivo
    • Definir almacenamiento (local vs nube)
    • Considerar versión futura con carga automática desde tiendas

Scenario
Steps
1. EMPRESARIOADMIN INVENTARIO accede al módulo de salida de productos. SALIDA DE PRODUCTOS
2. SYSTEM muestra la opción de cargar respaldo.
3. EMPRESARIOADMIN INVENTARIO selecciona un archivo Excel desde su dispositivo.
4. SYSTEM valida el tipo de archivo.
5. SYSTEM valida la estructura del archivo (columnas requeridas).
6. SYSTEM procesa el archivo y extrae productos y cantidades.
7. SYSTEM verifica los productos contra el inventario.
8. SYSTEM genera una vista previa de los datos procesados.
9. SYSTEM asocia temporalmente el archivo al proceso de salida.
10. EMPRESARIOADMIN INVENTARIO revisa la información mostrada.
11. SYSTEM continúa el flujo hacia la validación y registro de salida.
