DE INVENTARIO Y EMPLEADO, CONSULTAR INFORMACION DE LOS 5 ARTICULOS MÁS VENDIDOS (CON MAS SALIDAS DEL INVENTARIO CON MOTIVO DE SALIDA "VENTA") DEL STOCK EN EL MES Y LOS 5 MENOS VENDIDOS (PRODUCTOS CON MENOS SALIDAS DEL INVENTARIO CON MOTIVO DE SALIDA "VENTA"), CON MOTIVO DE SALIDA DE "VENTA" PARA NO TENER PRESENTE AQUELLOS QUE SALIERON POR DAÑO, HURTO, CONSUMO INTERNO U OTROS, SINO LOS SOLICITADOS POR CLIENTES EN COMPRAS.

Use Case Note
■ Workflow
    • // Write down briefly how user perform the work
    • El sistema va a mostrar las estadísticas de los productos en un diagrama de barras
    • Las estadísticas van a mostrar los 5 artículos con más rotación (más solicitados) y loa 5 con menos rotación (menos solicitados).
    • El sistema muestra las estadísticas de mayor rotación a menor rotación.
■ Business Logic
    • // Write down what user expect the system to react upon certain condition (e.g. low inventory alert level)
    • El usuario espera que al momento de ingresar la entrada o la salida de un producto, el sistema guarde la información para mostrar las estadísticas.
    • El usuario espera que el sistema le muestre las estadísticas de mayor rotación a menor rotación.
    • El usuario espera que el sistema le muestre las estadísticas de una forma fácilmente y entendible.
    • El usuario espera que el sistema organice los artículos por orden de rotación en el mes.
■ Decisions
    • // Write down the decisions made during the meeting (e.g. Must allow accessing from mobile devices)
    • Se decidio que los actores de rol Empresario, Admin. de Inventario y Empleado pueden ver las estadisticas
    • Se decidió que el sistema mostrará los 5 productos más vendidos (rotados) que al momento de registro de salida hayan sido los que más salieron por motivo de VENTA
    • Se decidió que el sistema mostrará los 5 productos menos vendidos (rotados) que al momento de registro de salida hayan sido los que menos han salieron por motivo de VENTA
    • Se decidió que el sistema va a mostrar las estadísticas no por categorías sino por productos
■ Follow-up
    • // Write down the items that should follow-up in the coming meeting
    • Que no solo pueda visualizar sino que por medio de las estadisticas pueda seleccionar el producto de su interes y este lo lleva al apartado de Inventario con toda la informacion de ese producto

Scenario
Steps
1. SYSTEM Guarda el registro de los artículos con mayor cantidad de entradas y salidas en el mes resgistradas con motivo "VENTA".
2. SYSTEM Crea un diagrama de barras para mostrar las estadísticas de los 5 artículos más rotados en el mes y los 5 menos rotados.
3. Cuando llegue un nuevo mes, se reinicia el ranking y empieza de nuevo el conteo para las gráficas.
4. if
    4.1. SYSTEM valida que hay mayoria o menoria de productos con registro de salida con motivos diferentes a "VENTAS":
    4.2. No los tiene en cuanta para realizar la inclusión a las estadisticas
   end if
