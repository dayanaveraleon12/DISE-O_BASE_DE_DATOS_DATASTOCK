Use Case Note
■ Workflow
    • // Write down briefly how user perform the work
    • El actor (Empresario, Admin. Inventario o Empleado) navega hacia el módulo de inventario en la plataforma.
    • El sistema despliega la lista de productos registrados con su información (producto, marca, presentación, stock, lote y vencimiento). En stock total se agrega la ubicacion (Tienda o Bodega).
    • El actor puede utilizar la barra de búsqueda y los filtros disponibles ( por ubicación, categoría, fecha de vencimiento, marca, etc.) para refinar la información mostrada y el filtro permite seleccion multiple
    • El sistema actualiza dinámicamente la lista de productos según los criterios de búsqueda y filtros aplicados.
    • El sistema despliega la lista de productos registrados, mostrando su categoría, cantidad y fecha de caducidad.
■ Business Logic
    • // Write down what user expect the system to react upon certain condition (e.g. low inventory alert level)
    • El sistema debe consultar y mostrar la información actualizada del inventario en tiempo real.
    • El sistema debe calcular automáticamente y mostrar el stock total de productos según el contexto (general o filtrado).
    • El sistema debe permitir búsquedas por texto (nombre, marca o lote).
    • El sistema debe permitir aplicar filtros combinados (por tienda/bodega, categoría, fecha de vencimiento, etc.).
    • Si ocurre un error en la carga de datos, el sistema debe mostrar un aviso al usuario de falla en la carga y permitir reintentar la operación.
■ Decisions
    • // Write down the decisions made during the meeting (e.g. Must allow accessing from mobile devices)
    • Todos los roles pueden visualizar el inventario completo.
    • Los filtros estarán disponibles visualmente junto a la barra de búsqueda para mejorar la experiencia de usuario.
    • Los filtros pueden aplicarse de forma simultánea (filtros combinados).
    • La información mostrada debe actualizarse en tiempo real al aplicar filtros o búsquedas.
■ Follow-up
    • // Write down the items that should follow-up in the coming meeting
    • Si ocurre un error en la carga de datos, el sistema debe permitir al usuario notificar el error en la plataforma.
    • Definir si se incluirán filtros avanzados adicionales (ej. rango de fechas, productos por vencer).

Scenario
Steps
1. EMPRESARIOADMIN INVENTARIOEMPLEADO selecciona la opción de ingresar al módulo de inventario.
2. SYSTEM Verifica los permisos del usuario.
3. SYSTEM Consulta la base de datos para obtener los productos registrados.
4. if Ocurre una falla de conexión o carga:
    4.1. Ir a Flujo Alternativo "Error en carga de datos"
   end if
5. SYSTEM Calcula el stock total de productos.
6. SYSTEM Muestra la lista de productos en pantalla.
7. EMPRESARIOADMIN INVENTARIOEMPLEADO ingresa un criterio en la barra de búsqueda (nombre, marca o lote).
8. SYSTEM Filtra los resultados según el criterio ingresado.
9. EMPRESARIOADMIN INVENTARIOEMPLEADO selecciona uno o más filtros (ej. tienda, categoría, fecha de vencimiento).
10. SYSTEM Aplica los filtros seleccionados.
11. if Si no encuentra datos relacionados con el filtro:
    11.1. Ir a Flujo Alternativo "Sin resultados en busqueda".
    end if
12. SYSTEM Actualiza la lista de productos en tiempo real mostrando solo los resultados filtrados.
13. SYSTEM Recalcula y muestra los totales de stock según los resultados filtrados.

Error en carga de datos
Steps
1. SYSTEM Detecta fallo en la consulta de datos.
2. SYSTEM Muestra mensaje de error al usuario.
3. EMPRESARIOADMIN INVENTARIOEMPLEADO puede reintentar la operación.

Sin resultados en búsqueda/filtros
Steps
1. SYSTEM No encuentra coincidencias.
2. SYSTEM muestra mensaje: "No se encontraron resultados".
3. EMPRESARIOADMIN INVENTARIOEMPLEADO puede modificar o limpiar los filtros.
