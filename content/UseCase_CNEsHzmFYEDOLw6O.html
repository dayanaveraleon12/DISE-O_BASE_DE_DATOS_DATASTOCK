Use Case Note
■ Workflow
    • // Write down briefly how user perform the work
    • El administrador de inventario o el empresario accede al módulo de salida de productos, carga un archivo (Excel) con el reporte de ventas enviado por empleados de las tiendas.
    • El sistema procesa el archivo, identifica productos y cantidades, y muestra un resumen.
    • El usuario valida la información, puede corregir o agregar productos, y finalmente confirma el registro de salida.
■ Business Logic
    • // Write down what user expect the system to react upon certain condition (e.g. low inventory alert level)
    • Debe permitir cargar archivos tipo Excel válidos
    • Procesa automáticamente los productos y cantidades
    • Verifica que los productos existan en el inventario
    • Valida disponibilidad de stock antes de confirmar
    • Permite edición manual antes de guardar
    • Registra la salida con trazabilidad (usuario, fecha, evidencia)
    • Si hay errores en el archivo: El sistema notifica inconsistencias (productos no encontrados, formatos inválidos, etc.)
■ Decisions
    • // Write down the decisions made during the meeting (e.g. Must allow accessing from mobile devices)
    • Tanto el administrador como el empresario pueden registrar salidas de productos
    • El archivo Excel será la evidencia principal obligatoria
    • El usuario siempre debe validar el análisis antes de guardar
    • Se permitirá corrección manual antes del registro final
■ Follow-up
    • // Write down the items that should follow-up in the coming meeting
    • Definir formato estándar del archivo Excel
    • Evaluar integración automática con tiendas en el futuro
    • Implementar alertas por diferencias entre archivo y stock
    • Validar si se requiere aprobación para grandes salidas

Scenario
Steps
1. EMPRESARIOADMIN INVENTARIO accede a la opción de "salida de productos".
2. SYSTEM muestra la opción para cargar archivo de evidencia.
3. EMPRESARIOADMIN INVENTARIO selecciona y carga el archivo Excel.
4. SYSTEM valida el formato del archivo.
5. SYSTEM procesa el archivo y extrae productos y cantidades.
6. SYSTEM compara los productos con el inventario existente.
7. SYSTEM muestra un listado preliminar de productos y cantidades detectadas.
8. EMPRESARIOADMIN INVENTARIO revisa la información mostrada.
9. EMPRESARIOADMIN INVENTARIO corrige, elimina o agrega productos manualmente si es necesario.
10. EMPRESARIOADMIN INVENTARIO confirma el registro de salida.
11. SYSTEM valida disponibilidad de stock.
12. SYSTEM registra la salida de productos.
13. SYSTEM descuenta las cantidades del inventario.
14. SYSTEM guarda el archivo como evidencia asociada.
15. SYSTEM muestra mensaje de confirmación exitosa de registro de saluda.

Stock insuficiente
Steps
1. SYSTEM detecta falta de stock
2. SYSTEM Muestra alerta
3. SYSTEM Cancela operación

Campos incompletos
Steps
1. SYSTEM detecta datos faltantes
2. SYSTEM Solicita completar información
3. SYSTEM Regresa al paso correspondiente
