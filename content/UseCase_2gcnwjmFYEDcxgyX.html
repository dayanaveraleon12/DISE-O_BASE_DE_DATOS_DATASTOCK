Use Case Note
■ Workflow
    • // Write down briefly how user perform the work
    • A usuarios previamente creados el sistema les permite ingresar
    • El usuario ingresa correo y contraseña
    • Y loguearse dando clic en el botón de iniciar sesión
■ Business Logic
    • // Write down what user expect the system to react upon certain condition (e.g. low inventory alert level)
    • El sistema valida que el campo para ingresar usuario tenga más de 8 caracteres
    • El sistema valida que el campo para ingresar contraseña tenga más de 8 caracteres escritos
    • Lo anterior para que se active el botón INICIAR SESION
■ Decisions
    • // Write down the decisions made during the meeting (e.g. Must allow accessing from mobile devices)
    • Se decidió que todos los actores previamente registrados pueden loguearse.
    • Se decidió que va a existir la opción de cambiar la contraseña en caso de olvidarla
■ Follow-up
    • // Write down the items that should follow-up in the coming meeting
    • El sistema podría notificar el ingreso de usuarios en otros dispositivos no autorizados a los de la empresa

Scenario
Steps
1. * El usuario ingresa a la plataforma y proporciona su correo electrónico y contraseña.
2. SYSTEM Consulta la base de datos para verificar la existencia del usuario y la validez de las credenciales.
3. * Una vez autenticado, el sistema identifica el rol asignado (EMPRESARIOADMIN INVENTARIOEMPLEADO).
4. SYSTEM Redirige al usuario al tablero principal (dashboard) correspondiente a sus permisos.
5. Las contraseñas deben cumplir con una complejidad de 8 carácteres, incluyendo valores alfanuméricos y carácteres especiales.
6. SYSTEM Debe registrar cada intento de acceso exitoso
7. * El acceso a los módulos de "Productos" o "Registro" depende estrictamente del rol verificado durante el login.
8. if Si el correo no existe, el sistema muestra un aviso que indica se debe contactar con el Empresario.
   end if
9. if * Si la contraseña no coincide, el sistema genera un aviso indicando "Contraseña incorrecta; cambia tu contraseña".
    9.1. CAMBIAR CONTRASEÑA
   end if
10. if * Si el usuario selecciona "olvidé mi contraseña", el sistema inicia el flujo de recuperación enviando un código de 6 dígitos al correo.
    10.1. VALIDAR CONTRASEÑA
    end if
11. * El sistema valida qué opciones/ventanas mostrar basándose en si el usuario es Empresario y/o Administrador (con poder de edición) o un empleado (solo visualización, envio de listas).
12. VALIDAR ROLES

Preconditions El usuario debe estar registrado previamente en la base de datos con credenciales válidas. Caso de uso previo: UC03 (Crear Usuarios) o UC53 (Cambiar Contraseña).
Post-conditions El sistema otorga acceso y redirige al usuario al dashboard según su rol asignado. Caso de uso posterior: UC20 (Consultar Inventario) o UC02 (Recibir Notificaciones Ingreso).
