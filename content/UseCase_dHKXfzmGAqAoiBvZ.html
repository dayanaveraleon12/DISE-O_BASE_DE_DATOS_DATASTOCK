Use Case Note
■ Workflow
    • // Write down briefly how user perform the work
    • El administrador de inventario o el empresario, después de cargar y validar el archivo de respaldo (Excel) y revisar los productos a registrar, ingresa una descripción general de la salida, explicando el contexto del movimiento (por ejemplo: ventas del día por tiendas).
    • Finalmente, confirma el registro y el sistema guarda la descripción junto con la salida.
■ Business Logic
    • // Write down what user expect the system to react upon certain condition (e.g. low inventory alert level)
    • Permite ingresar una descripción asociada a toda la salida (no por producto individual)
    • Asocia la descripción al registro generado desde el archivo Excel
    • Permite consultar la descripción en el historial de movimientos
    • Mantiene la trazabilidad junto con usuario, fecha y evidencia (archivo)
    • Puede exigir la descripción como campo obligatorio antes de guardar
    • Si la descripción es obligatoria (en caso de que el motivo de salida sea: daño, consumo interno, vencimiento):
        ♦ El sistema no permite continuar el registro sin este campo
■ Decisions
    • // Write down the decisions made during the meeting (e.g. Must allow accessing from mobile devices)
    • La descripción será obligatoria para registrar la salida solo en casos especificos (daño, consumo interno, vencimiento)
    • Se manejará como descripción global del movimiento (no por producto)
    • Se permitirá texto libre
    • Debe visualizarse en el historial y reportes
    • Aplica tanto para administrador como empresario
■ Follow-up
    • // Write down the items that should follow-up in the coming meeting
    • Definir longitud máxima de la descripción
    • Evaluar uso de plantillas (ej: “Ventas día X tienda Y”)
    • Analizar si se requieren descripciones por producto en el futuro
    • Integrar con reportes de auditoría

Scenario
Steps
1. EMPRESARIOADMIN INVENTARIO selecciona la opción de "Registrar Salida de Inventario".
2. SYSTEM solicita la carga del archivo de respaldo (Excel).
3. EMPRESARIOADMIN INVENTARIO carga el archivo y el sistema valida su estructura y contenido.
4. SYSTEM muestra la lista de productos y solicita seleccionar el Motivo de Salida.
5. EMPRESARIOADMIN INVENTARIO selecciona un motivo que requiere descripción (Daño, Consumo interno o Vencimiento).
6. SYSTEM habilita el campo "Descripción de Salida" y muestra un indicador de obligatoriedad.
7. EMPRESARIOADMIN INVENTARIO ingresa la descripción general explicando el contexto del movimiento.
8. EMPRESARIOADMIN INVENTARIO confirma el registro de la salida.
9. valida que el campo de descripción no esté vacío.
10. SYSTEM guarda el registro de salida asociando la descripción, el usuario, la fecha, la hora y el archivo cargado.
11. SYSTEM muestra un mensaje de confirmación de registro exitoso.

Descripción Opcional
Steps
1. Motivo diferente: En el paso 5 del Flujo Básico, si el actor selecciona un motivo distinto (ej. Ventas, Traslado), el sistema habilita el campo "Descripción de Salida" como opcional.
2. Omisión de descripción: El actor puede optar por dejar el campo vacío y proceder al paso 8. El sistema permitirá el guardado sin generar errores.

Campo Obligatorio Vacío
Steps
1. En el paso 9 del Flujo Básico, el sistema detecta que el motivo es (Daño, Consumo interno o Vencimiento) pero el campo "Descripción" está vacío.
2. El sistema bloquea el registro y resalta el campo de descripción en rojo.
3. El sistema muestra el mensaje de error: "La descripción es obligatoria para el motivo de salida seleccionado".
4. El flujo regresa al paso 7 del Flujo Básico para que el actor corrija la entrada.
