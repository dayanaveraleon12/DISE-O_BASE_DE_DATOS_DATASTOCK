Use Case Note
■ Workflow
    • // Write down briefly how user perform the work
    • El sistema va a analizar diariamente las fechas de caducidad ingresadas en la creación de lotes.
    • Si el sistema llega a detectar que una fecha está próxima a vencer en un determinado periodo de tiempo (Ejemplo: un artículo se vence en 3 días), el sistema va a enviarle una notificación indicando cuál es el artículo próximo a vencer.
    • La notificación debe decir: "El artículo (nombre del artículo), va a vencer en un tiempo de (número de días).
■ Business Logic
    • // Write down what user expect the system to react upon certain condition (e.g. low inventory alert level)
    • El usuario espera que se genere una notificación de acuerdo a la fecha de caducidad de un artículo.
    • El usuario espera que en la notificación se le indique cuántos días quedan para que caduque el artículo correspondiente.
    • El usuario espera que ésta notificación se genere solo en la plataforma DataStock
■ Decisions
    • // Write down the decisions made during the meeting (e.g. Must allow accessing from mobile devices)
    • Se decidió que la notificación se generara en cada inicio de sesion del Empresario, Admin. de Inventario y Empleado (para que vea la informacion oportunamente)
    • Cada notificacion queda registrada en el historial de notificaciones (Campanita)
    • Se generan notificaciones de caducidad 1 mes antes, 15 días antes y el mismo día si aun esta disponible en Stock
■ Follow-up
    • // Write down the items that should follow-up in the coming meeting
    • N/A

Scenario
Steps
1. SYSTEM identifica y consulta diariamente las fechas de vencimiento de los lotes ya creados.
2. SYSTEM genera esta notificacion de caducidad validando productos en Stock que esten en rangos de vencimiento de: 1 mes antes, 15 días antes ó el mismo día sí aun esta disponible en Stock
3. if Si hay un lote próximo a vencer en los rangos ya mecionados:
    3.1. SYSTEM genera un mensaje de alerta en cada inicio de sesión del EMPRESARIOADMIN INVENTARIOEMPLEADO con datos como: producto, lote, fecha de caducidad y días restantes y tambien lo almacena en el historial de notificaciones.
    3.2. if Si no hay un lote próximo a vencer:
        3.2.1. SYSTEM no generará un mensaje de alerta.
         end if
   end if
4.
