Use Case Note
■ Workflow
    • // Write down briefly how user perform the work
    • El actor (Administrador de Inventario o Empresario) accede al módulo “Historial” desde la barra de navegación del sistema.
    • El sistema redirige a la vista de historial de movimientos.
    • El sistema muestra una lista de registros de modificaciones realizadas en el inventario.
    • El actor puede visualizar cada registro con información resumida (usuario, acción, producto, fecha y hora).
    • El actor selecciona un registro específico para ver el detalle completo del movimiento como el desglose de los productos con su marca y cantidad.
■ Business Logic
    • // Write down what user expect the system to react upon certain condition (e.g. low inventory alert level)
    • El sistema muestra los registros ordenados por fecha y hora descendente (más recientes primero).
    • Cada registro debe contener:
        ♦ Usuario que realizó la acción
        ♦ Tipo de acción (entrada, salida)
        ♦ Producto afectado
        ♦ Fecha y hora exacta
    • Para movimientos de inventario:
        ♦ Cantidad anterior
        ♦ Cantidad nueva
    • El sistema permite filtrar los movimientos por fecha mensual.
    • El historial es solo de lectura (no editable).
    • El sistema debe garantizar la integridad y trazabilidad de los registros.
    • Tener indicadores visuales por tipo de acción como entradas(color verde) y salidas(color rojo).
■ Decisions
    • // Write down the decisions made during the meeting (e.g. Must allow accessing from mobile devices)
    • El historial será accesible tanto para el Administrador de Inventario como para el Empresario.
    • Se implementa como mecanismo de auditoría y control del sistema.
■ Follow-up
    • // Write down the items that should follow-up in the coming meeting
    • Implementar exportación del historial (PDF/Excel).

Scenario
Steps
1. El EMPRESARIOADMIN INVENTARIO accede al sistema.
2. Selecciona la opción “Historial” en el menú principal.
3. SYSTEM consulta los registros almacenados.
4. SYSTEM muestra la lista de movimientos ordenados por fecha descendente.
5. Se visualizan los registros disponibles.
6. El EMPRESARIOADMIN INVENTARIO (opcionalmente) selecciona un registro para ver más detalles.
7. SYSTEM muestra la información detallada del movimiento seleccionado
8. if EMPRESARIOADMIN INVENTARIO usa el filtro de busqueda para filtrar los movimientos por fecha mensual.
    8.1. SYSTEM carga los movimientos del rango de la fecha seleccionados
   end if

Error al cargar datos
Steps
1. if Si Ocurre un error al consultar la base de datos por fallas externas, entonces:
    1.1. SYSTEM muestra un mensaje de error.
    1.2. SYSTEM permite reintentar la carga.
   end if
